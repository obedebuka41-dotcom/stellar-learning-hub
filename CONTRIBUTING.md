# Contributing To Stellar Learning Hub

Thanks for your interest in contributing. Stellar Learning Hub is being prepared as an open-source education project for Stellar developers.

## Before You Start

- Read the issue description fully.
- Ask for clarification before starting if the acceptance criteria are unclear.
- Keep your pull request focused on one issue.
- Do not include unrelated refactors in bounty work.

## Local Setup

```bash
npm install
npm run dev
```

Open `http://localhost:3000` to view the app.

## Pull Request Guidelines

- Link the pull request to the issue it solves.
- Explain what changed and why.
- Include screenshots or short recordings for UI changes.
- Mention any known limitations.
- Make sure the app builds before requesting review.

## Issue And Bounty Expectations

For bounty issues, maintainers should review work against the acceptance criteria in the issue. A contribution is considered complete when:

- The requested feature or content is implemented.
- The acceptance criteria are met.
- The code is readable and fits the existing project structure.
- The pull request has enough detail for review.

## Code Style

- Use TypeScript for app code.
- Prefer small reusable components.
- Keep learning content clear and beginner-friendly.
- Avoid adding large dependencies unless the issue calls for them.

## Community

Be respectful, patient, and clear. This project is meant to help developers learn, so explanations and documentation matter as much as the code.
